# 🎯 기획 의도 및 요구사항 정의서

## 1. 설계 목표
- **사용자 중심의 학습 경험**: 단순히 강의를 보는 것을 넘어, 커뮤니티(`post`, `comment`)와 강사 팔로우(`like_teacher`)를 통해 소통하는 플랫폼 지향.
- **AI RAG 최적화**: 수강평(`review`) 데이터를 독립적으로 관리하여, AI Agent가 특정 강좌의 장단점을 빠르게 파악하고 사용자에게 추천할 수 있는 구조 구축.

## 2. 핵심 로직 고민 (User's Thoughts)
- **복합적인 관심도 측정**: 영상 하나에 대한 '좋아요'(`like_lecture`)와 강사에 대한 '좋아요'(`like_teacher`)를 분리하여 사용자의 취향을 다각도로 분석함.
- **유연한 수강 관리**: `listening_course`를 통해 사용자와 강좌를 연결함으로써, 수강 기간 내에만 강의를 볼 수 있는 권한 관리 로직의 기반을 마련함.
