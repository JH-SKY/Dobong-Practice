# 랜덤 조짜기 프로그램
### 1. 기획 의도 (Why)
- 교육생 명단을 무작위로 섞어 공정하게 조를 나누기 위해 제작함. 단순히 나누는 기능을 넘어 실무에서 발생할 수 있는 예외 상황을 처리하는 연습을 목적으로 함.
### 2. 설계 및 구조 (Architecture)
- 함수 기반 설계: 조 나누기 로직(`make_group`)과 파일 저장 로직(`save_to_file`)을 분리하여 유지보수가 쉽게 구성함.
- 데이터 흐름: 명단 리스트 입력 -> 무작위 셔플 -> 지정된 인원수만큼 슬라이싱 -> 딕셔너리 저장 -> txt 파일 출력.
### 3. 핵심 로직 및 기술 선택 이유 (Core Logic & Selection)
- `random.shuffle`: 파이썬 내장 라이브러리를 사용하여 별도의 설치 없이 빠르게 무작위 기능을 구현함.
- `f-string`: 가독성 높은 문자열 포맷팅을 위해 사용함.
- `try-except`: 파일 입출력 시 발생할 수 있는 외부 환경 에러를 방지함.
### 4. 트러블슈팅 (Troubleshooting)
- 에러 메시지: `TypeError: make_group() got an unexpected keyword argument 'size'`
- 원인 분석: 함수를 정의할 때는 매개변수명을 `members_per_group`으로 지었으나, 호출할 때 `size`라는 이전 변수명을 그대로 사용함.
- 해결 과정: 함수 호출부의 키워드 인자명을 정의부와 일치하도록 수정하여 인자 전달 오류를 해결함.
### 5. 업데이트 기록 (Changelog)
- v1.0 : 초기 설계 및 리스트 셔플 기능 구현
- v1.1 : 예외 처리(빈 리스트 체크), 파일 저장 기능 추가 및 실무형 변수명으로 리팩토링
### 6. 실무 관점의 참견 (Insight)
- 실제 서비스라면 명단이 1,000명이 넘어갈 수 있으므로, 메모리 효율을 위해 전체를 리스트로 읽기보다 제너레이터를 활용하는 방식도 고려해볼 법함. 또한, 조 배정 결과를 DB에 저장하여 이력 관리가 필요해 보임.
### 7. 성장 포인트 (Retrospective)
- 변수명을 지을 때 나만 아는 `lst`가 아니라 `assigned_groups`처럼 역할이 드러나는 이름을 쓰는 것이 협업의 시작임을 배움. 함수 호출 시 인자명 일치의 중요성을 에러를 통해 체득함.
